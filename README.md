## Overview

This ArcGIS toolbox allows for raw images from an RTK DJI drone flight to be directly georeferenced. Python script generated by Claude and edited by the authors uses onboard GPS and IMU metadata to project each image to its geographic position. The workflow begins with inputting raw images and a takeoff offset, defined by the difference between the water surface level and drone takeoff point. Within the script, metadata extraction parses EXIF and XMP headers for GPS coordinates, relative altitude, and gimbal orientation, which are then projected into the Universal Transverse Mercator coordinate system. The script calculates the Ground Sampling Distance and rotates images based on gimbal yaw. Each processed image is saved with a corresponding world file (.pgw) that tells GIS software  where to place the image (location of the top-left corner, pixel size, and rotation factor). It also generates a projection (.prj) file, which defines the coordinate system for that specific image. Following georeferencing, the images can be imported into an ArcGIS Pro Mosaic Dataset. This tool was built around the DJI Mavic 3 Multispectral system.
